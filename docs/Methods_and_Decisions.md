## 1. Pipeline overview

`CSI capture → webcam auto-labeling → 2 s windows → feature extraction →
per-environment calibration → Random Forest → evaluation`

All code is in the repo: firmware (`firmware/`), capture (`collect_av.py`,
`log_csi.py`), dataset/features (`csi_dataset.py`, `phase1_av.py`,
`presence_model.py`), analysis (`coverage_analysis.py`, `model_stats.py`).

## 2. Feature dictionary

Per 2 s window of CSI amplitude `|CSI| = sqrt(I^2 + Q^2)`, restricted to the
**~52 active subcarriers** (DC + guard nulls dropped — see §3 mask). Vector
length **168 = 52×3 + 8 + 4**.

| Group | Count | Definition | Rationale |
|---|---|---|---|
| `dev_sc[i]` | 52 | per-subcarrier mean amplitude minus the site's empty baseline, divided by baseline `(mean−base)/base` | presence = how far the static channel is from "empty" |
| `std_sc[i]` | 52 | temporal std of each subcarrier over the window | motion magnitude (level-invariant) |
| `motion_sc[i]` | 52 | fraction of FFT power in 0.5–5 Hz per subcarrier | gross motion / Doppler band |
| `dev_mean,max,l2` | 3 | aggregate stats of the deviation vector | overall presence strength |
| `one_minus_cos` | 1 | `1 − cosine(window-mean shape, empty shape)` | shape change vs empty, level-free |
| `std_mean,max` | 2 | aggregate of per-subcarrier std | overall motion |
| `motion_mean,max` | 2 | aggregate of motion-band fraction | overall motion-band energy |
| `breath_*` (4) | 4 | 0.15–0.5 Hz band fraction (mean/max) and peak-sharpness (mean/max) over a ~20 s context | slow low-frequency fluctuation (helps still occupants). NB: framed as "low-frequency fluctuation," we are **not** claiming breathing-rate estimation |

Amplitude only; **phase is deferred** (ESP32 raw phase is corrupted by CFO/SFO/random offsets and needs sanitization — listed as an ablation to add).

## 3. Heuristics & parameters (value + why)

| Parameter | Value | Rationale |
|---|---|---|
| Window length | 2 s (~200 packets) | balance: long enough to measure variation, responsive enough for presence; longer (3/4/6 s) tested, no gain |
| Window hop | 1 s (50% overlap) | one decision/sec |
| `MIN_PKTS` per window | 100 | reject windows thinned by packet loss |
| Label purity gate | ≥0.7 | drop windows straddling an activity transition |
| Active-subcarrier mask | mean amp > 0.1 × median | data-driven removal of DC/guard nulls (≈52/64), ordering-agnostic |
| Motion auto-calibration (labeling) | "moving" if positional wander over ~1.2 s > (25th-pctile of recent wander)×4 + base | self-calibrates to each subject/camera; no hand-set threshold |
| Empty calibration (mean) | subtract per-site empty baseline | removes site fingerprint |
| Empty calibration (scale, `zcal`) | z-score features vs that site's empty windows | fixes cross-site deviation-magnitude mismatch; prevents held-out collapse |
| Classifier | RandomForest, 400 trees, `class_weight=balanced` | robust on modest tabular data, interpretable |
| Decision threshold | 0.5 | tuning on train folds tested → hurt; kept simple |

## 4. Model statistics (summary; full in `model_stats_report.md`)

- **Dataset:** 1,944 windows · 12 takes · 3 link geometries · 1 subject (preliminary).
- **Per-class means (interpretable features)** behave physically: deviation (`dev_*`)
  is lowest for *empty*, highest for *still/moving*; `motion_*` highest for *moving*;
  low-freq (`breath_*`) higher when occupied.
- **Most important features (RF):** `one_minus_cos`, `breath_fracmax`, `dev_mean`,
  `dev_l2` — i.e., the calibration/deviation and low-frequency features carry the signal.
- **Unsupervised separability is poor:** KMeans(k=3) vs labels **ARI 0.05, NMI 0.05,
  silhouette 0.17**; PCA(2) explains 56% variance. The classes **overlap** in feature
  space and t-SNE structure tracks sessions/environments more than classes →
  supervised learning is required, and per-site calibration is essential.

## 5. Decisions log (chronological, with rejected alternatives)

- **Firmware connection:** removed a forced custom STA MAC that blocked association on the C3; RX now **auto-locks** to the first station. *(Rejected: manual MAC matching.)*
- **AV capture drops:** YOLO starved the serial reader → 12–56% CSI loss. **Fix: 1 MB serial RX buffer** → 0 drops. *(Rejected: lowering frame rate.)*
- **Multi-ESP topology (planned):** moving to **1 TX broadcast + N RX sniffers (promiscuous)**, all on one PC clock. *(Rejected: N independent pairs — harder to time-align.)*
- **Labels:** canonical {empty, still, moving}; webcam + YOLO11-pose auto-labeling; loader **requires a matching JSON sidecar with a canonical label** after renamed files once poisoned the set (walking data labeled "empty"). Activity going forward uses **scripted cues** (robust to table occlusion that defeated pose sit/stand).
- **Motion labeling:** centroid-speed missed walking when the subject fills the frame → switched to **positional-wander with auto-calibrated threshold**.
- **Features/calibration:** amplitude-only; empty-baseline subtraction (mean) → added **scale calibration (zcal)** after a held-out geometry inverted; added **low-frequency features** (improved cross-geometry +5 pts).
- **Evaluation:** **leave-one-session / -room / -subject-out**, never random-window splits (random split read 99% vs ~73% held-out — documented to justify the methodology).
- **Rejected tweaks (measured, no gain):** decision-threshold tuning (overfit, pushed to "occupied"); temporal smoothing 5 s (wash); window length 3/4/6 s (2 s best).

## 6. Known limitations (stated up front)

- Preliminary numbers are **single-subject**, two room types, **old associated-link firmware**; will be regenerated on the rigorous **broadcast/sniffer, multi-orientation, multi-subject** dataset.
- CSI is hardware/firmware/environment-specific; results are for ESP32-C3 HT20 LLTF.
- Motion sensing is coverage-limited by a single link (documented dead zones).
- "Low-frequency" features are not validated as true breathing-rate estimates.
